Each of the following statements has a syntax mistake. Write the statements properly and give a sentence saying what was wrong. (Cut and paste the correct statement from R, along with any output that R gives and your sentence saying what was wrong in the original.)

Here's an example:

QUESTION: What's wrong with this statement?
a = fetchData(myfile.csv)

ANSWER: It should be a =fetchData("myfile.csv") The file name is a character string and therefore should be in quotes. Otherwise it's treated as an object name and there is no object called myfile.csv.

Now for the real thing. Say what's wrong with each of these statements for the purpose given:

  • seq(5;8) to give [1] 5 6 7 8

Nothing is wrong.
Use a comma instead of a semi-colon to separate arguments: seq(5,8).
It should be seq(5 to 8).

ANSWER: The semicolon is not a valid separator between arguments. The statement should be seq(5,8).

  • cos 1.5 to calculate the cosine of 1.5 radians

ANSWER:The parentheses have been left out. It should be cos(1.5).

  • 3 + 5 = x to make x take the value 3 + 5

ANSWER: Assignment statements should have the object name on the left: x = 3 + 5

  • sqrt[4*98] to find the square root of 392

ANSWER: Use parentheses, not square brackets, to invoke an operator: sqrt(4*98).

  • 10,000 + 4,000 adding two numbers to get 14,000

ANSWER: Don't punctuate numbers with commas, just the digits: 10000 + 4000.

  • sqrt(c(4,15,25,36)) = 4 intended to give [1] FALSE TRUE FALSE FALSE

ANSWER: Only a single = was used, the assignment operator rather than equality. It should be sqrt(c(4,16,25,36)) == 4

  • fruit = c(apple, berry, cherry) to create a collection of names [1] "apple" "berry" "cherry"

ANSWER: The character strings have to be quoted to distinguish them from object names. It should be fruit = c( "apple", "berry", "cherry").

  • x = 4(3+2) where x is intended to be assigned the value 20

ANSWER: The multiplication operator needs to be specified explicitly and not implicitly as in algebraic notation. The statement should be x = 4*(3+2).

  • x/4 = 3+2 where x is intended to be assigned the value 20

ANSWER: Only an object name can be on the left side of the assignment operator. It should be x = 4*(3+2)
